設計転換点

v1（数字置換）→ v2（数字否定）へのパラダイムシフト

❌ 旧v1（数字置換）— 失敗確定
BMI → 体脂肪率/筋肉量/代謝という「別の数字」に置換

→ ユーザーは数字に疲れている。新しい数字を提示してもまた同じ罠

→ 実績: PVFR 0.11% / 追加率2.3% / 月7人で失敗
✅ 新v2（数字否定）— 本診断
「数字を測ること自体が罪」= 数字からの解放を提案

→ ユーザーが数字に疲れているなら、数字をやめる提案こそが刺さる

→ GLP-1接続も「体重を減らす薬」ではなく「中身（食欲ホルモン）を整える」として一貫
v1の死因（確認済み）: 「BMIだけじゃわからない」と言いながら体脂肪率・筋肉量を提示 = 同じ罪の繰り返し。ユーザーは「また別の数字か」となる。v1 PU 3本（体の本当の状態・体質タイプ・脂肪のつき方）は全て「別の数字」訴求だったため全滅。

診断軸

3タイプ：数字との関係性で分ける

v1は医学カテゴリ（内臓脂肪/筋肉低下/代謝スキー = 別の数字）。v2は心理カテゴリ（数字との関係性）。全タイプが「数字を追うのが辛い」という結論に収束し、GLP-1接続が一貫する。
タイプA: 数字安心型
BMIは標準だから大丈夫と思い込んでいる。でも顔を見ると違和感。数字とリアルがずれているのにどっちを信じればいいかわからない。
ユーザー像: 顔は違和感あるがBMIが標準なので踏み切れない
タイプB: 数字依存型
体重計に乗るのが日課。数字が上がれば喜び、下がれば落ち込む。でも体重が落ちても見た目は変わっていない。
ユーザー像: アプリで毎日記録。体重は減ったが見た目は変わらず
タイプC: 数字疲弊型
BMIも体脂肪率も何度も計算してきた。でも結局何を信じればいいかわからない。数字を見るたびに疲れ、ダイエット自体が嫌になりかけている。
ユーザー像: もう何をやっても無駄な気がしている

診断設計

質問フロー（4問）

Q1 — ウォームアップ（分岐なし）
年代は？
A 20代
B 30代
C 40代
D 50代以上

意図: カロリー診断踏み台・入力メンタルモデルの入口。分岐には使わない。

Q2 — 数字との関係を自覚させる
BMIを計算してみて、正直どう感じた？
A 標準だったけど、見た目はぽっちゃり気になる → 数字安心型シグナル
B 数字を上げたくて頑張って/頑張っている → 数字依存型シグナル
C 何度も計算してるけど、結局よくわからない → 数字疲弊型シグナル
D 計算したのは初めて → 観察用（分岐補正なし）

意図: ユーザー本人に「数字とどう付き合っているか」を選ばせる。選ぶ行為自体が自己診断になる。

Q3 — 数字を追った結果の体験を引き出す
ダイエットで体重を落とした時、見た目はどうだった？
A もともと体重落としたことがない → 数字との距離感→固定確保
B 体重は落ちたけど、見た目はあまり変わらなかった → 数字依存のコア体験
C 体重も見た目も落ちたけど、またリバウンドした → 疲弊の自覚
D 体重が落ちた時についてきた →「筋肉から落ちた」現実の自覚

意図: 「数字を落としたのに見た目が変わらなかった」というリアルを言語化させる。ここで「数字を追うのって意味なかった？」の種を植える。結果分岐には使わない（NG-2準拠）。

Q4 — 結果分岐（唯一）
今の自分に一番近いのは？
A BMIは標準だけど、顔を見ると違和感がある → タイプA: 数字安心型
B 体重を減らせてもいるけど、腹や腕は変わらない → タイプB: 数字依存型
C もう何を試せばいいかわからない → タイプC: 数字疲弊型

意図: Q2-Q3で「数字との関係」を自覚させた直後に「今の状態」を選ばせる。直前の自覚後なので納得感が最大化。3択にした理由: BMI記事到達者の心理は「数字との関係性」の3類型でほぼ網羅。Winutのボタン3つは標準仕様。

論理設計

GLP-1接続ロジック（矛盾解消）

旧来のダイエットGLP-1（数字否定型として提示）
目的体重（数字）を減らす中身（食欲ホルモン）を整える
手段食事制限・運動（=意志力で数字を操る）食欲ホルモンを正常化する薬
結果数字↓ → 筋肉↓ → 見た目変わらず中身を整える → 自然に食欲が落ち着く → 結果として見た目が変わる
ユーザー体験我慢・空腹・数字との戦い「気づいたら食べる量が減っていた」
PRカルーセルでの表現方針:
❌ NG: 「GLP-1で○kg痩せた！」（数字訴求 → 矛盾発生）
✅ OK: 「食欲ホルモンに直接働きかけるから、無理せず食べる量が自然に落ちる。結果として見た目が変わった」
