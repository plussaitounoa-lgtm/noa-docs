Chapter 1　前提知識
順位施策とは何か
一言で言うと
狙ったKWで検索上位に
表示させること
上位に出ないと、そのKWで検索したユーザーが記事に来られない
収益との接続
上位表示 → 流入増加 → CV記事へ送客 → 収益
順位が上がらない = 流入が増えない = 収益の伸び代がない
仕事の中での唯一の判断軸
「この改修でメインKW・
サブKWの順位が上がるか」
これだけで判断する。以下は目的に含まれない：
✗ CTRを上げる
✗ CVRを上げる
✗ 見た目・読みやすさを改善する
💡 ただし
CVRへの影響は常に考慮する。「順位を損なうアクションはしない」が鉄則。両立できない場合は順位を優先する。

独自性（So what）とは
よくある誤解
✗ 独自性ではない
「競合にない珍しい情報を仕入れること」
「独自調査・アンケートデータ」
← これは一次情報であって独自性の一形態ではあるが、本質ではない
✓ 独自性の本質
「この記事を読んだユーザーが次に何をすべきか、答えを出すこと」
＝ So what（だからどうする）を書くこと
具体例（AGAの治療法）
✗ 事実の羅列で終わる

AGAには内服薬・外用薬・植毛
の3つの治療法があります。
内服薬はフィナステリド・デュタ
ステリド・ミノキシジルがあり...

（「で、どれ選べばいい？」がない）
✓ So what まで出す

AGAには内服薬・外用薬・植毛の
3つの治療法があります。
＜各治療法の説明＞

まだ進行が浅い段階なら内服薬から始めるのが費用・効果のバランスが良い。
記事のゴールに直結する
ノウハウ記事のゴールは
「ユーザーが意思決定できる状態」。

So whatのないH3は、情報を渡しただけでゴールを達成していない。

「考えさせたら負け」。判断まで出しきる。
判断の問い
このH3を読んだユーザーは次に何をすべきかわかるか？　わからなければSo whatが足りない。

外部リンク（一次ソース） 厚生労働省・論文・製薬会社の公式情報にリンクを貼る
数値・統計への引用 「〇〇という研究によれば」「〇〇省の調査では〇%」と出典を明示する

根拠なし断言・最上級表現・効果の保証はNG。医療広告ガイドラインにも抵触する。

独自性 = So what（ユーザーへの結論）
E-E-A-T = 根拠・信頼性の担保

別の概念。両方必要。

Chapter 3　作業フロー
PHASE 1 前に ― GSCデータの読み方
GSCでわかること
「この記事が今、どのKWでGoogleに評価されているか」を読む。
狙ったKWで評価されていない＝KWがズレている。
見るべき指標と判断ポイント
指標何を見るか
クエリ 表示回数上位25〜30件を3〜5グループに分類する。意図グループが揃うと「何で評価されているか」がわかる
上位・0クリック 順位1〜10位なのにクリック数が0 → そのクエリの意図が記事で論点化されていないサイン
KWズレ 表示回数上位にメインKWが含まれていない → 別意図で評価されている。社員に報告・判断をもらう
CTRの目安
1位：約14%
2位：約7%
3位：約4%

これを大幅に下回る場合、タイトル・構成のどちらかに問題がある。
🚫 自分で判断しないこと
KWズレがあっても、自分でKWを切り替える判断をしない。必ず社員に報告して判断をもらう。サイト全体の方針に関わる。
Chapter 3　作業フロー
PHASE 2 ― やり切り診断の判断ポイント
診断の前に：PAA・関連KW・競合3記事のURLをユーザーに依頼してから始める
#診断軸「×」のとき何が起きているか
① 理解フロー順か 定義前に選び方が来ている → 前提知識がないユーザーが途中で離脱する
② クエリが論点化されているか H2/H3はあるが意図と噛み合っていない → Googleに「答えている」と評価されない
③ 競合と同水準の情報量か 競合が数値・手順で書いているのに自社が概要レベル → 情報深度で負けている
④ E-E-A-Tの担保があるか 根拠なし断言・引用元なし → 競合に対して信頼性で負けている
⑤ 独自性があるか 競合と同じ情報を並べただけ → 「同じ記事が複数ある」状態。上位に出る理由がない
⑥ 各H3にSo whatがあるか 事実の羅列で終わっている → ユーザーが「だから何？」で離脱する
△・×の書き方
「なんとなく薄い」は根拠ではない。

「競合の〇〇H2に対して、自社の〇〇H3は数値がなく概要レベルで終わっている」まで書く。
次のステップ判断
H2/H3の追加・削除・順番変更が必要
→ PHASE3（改修設計）へ

中身の情報補完・根拠追加だけ
→ PHASE5（出力）へ直接

Chapter 3　作業フロー
よくあるNG ― 判断ミスのパターン
GSCを見ずに「なんとなく構成が悪そう」で作業を始める
クエリ分析なしに競合を見ても、何が足りないかの判断軸がない。必ずGSCから入る。
KWズレを自分でKW切り替えに進める
KWスイッチはサイト全体の方針に関わる。必ず社員に報告・判断をもらってから動く。
△・×に「なんとなく薄い」と書く
根拠なし判定はフィードバックにならない。競合のどこと比べてどこが薄いか、まで書く。
So whatを「まとめ」の1行だけで書く
各H3の末尾にSo whatが必要。「このH3を読んだユーザーは次に何をすべきかわかるか」で確認する。
複数フェーズをまとめて進める
一気に出しても最初の分析がズレていたらやり直しになる。各フェーズで立ち止まって判断してから次へ進む。
「順位施策中にCVR改善も一緒にやる」
施策の軸を混ぜると何が効いたか判断できなくなる。順位施策中は順位のみに集中する。
