基本の流れ

操作のたびに git pull → 操作 → git push を守る。

# 1. リポジトリに移動
cd ~/OneDrive/デスクトップ/project-noa/line-team

# 2. 最新を取得（毎回必須）
git pull

# 3. manage.py で操作
python3 manage.py <コマンド>

# 4. pushして同期（tasks.json だけ add する）
git add data/tasks.json
git commit -m "update: 内容 (Noa)"
git push
💡 git add する前に git status で差分確認。想定外のファイルが含まれてたら止まる。

コマンド一覧

確認系

# 自分のタスクだけ確認
python3 manage.py list --assignee Noa

# 全タスク確認
python3 manage.py list

# タスクの詳細を見る
python3 manage.py show t5

更新系

# ステータス変更（todo / in_progress / watching / done）
python3 manage.py update t5 --column done

# メモを追加
python3 manage.py note t5 "作業メモ"

# 担当者を変更
python3 manage.py update t5 --assignee Ren

追加系

# 自分のタスクを追加
python3 manage.py add --title "タイトル" --purpose "目的・ゴール状態" --assignee Noa --created-by Noa

# Renにタスクを振る
python3 manage.py add --title "タイトル" --purpose "目的" --assignee Ren --created-by Noa

列（column）の意味

列意味

in_progress作業中
watchingデータ蓄積待ち・効果検証中
done完了

ルール

  • 操作前に必ず git pull
  • 操作後は必ず git push（data/tasks.json だけ add）
  • 自分のタスクだけステータス変更する。他人のは触らない
  • needs_approval: true のタスクはRenの承認が必要。勝手に完了にしない
