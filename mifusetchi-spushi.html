このスプシの目的

SEO記事への LINE PU（ポップアップ）・バナー の設置漏れを一覧で検知するツール。
記事にPUが入っていない = 訪問者がLINEに誘導されていない = 友だち追加機会がゼロになっている。
月間クリック数に PVFR（友達追加率）0.5% をかけた見込み人数を算出し、優先度をつけて管理する。

データソース：friend_data_cache.csv（LINEのタグ別友だち追加ログ）＋ Google Search Console（月間クリック数）
更新タイミング：月1回 or PU設置後に手動実行

⚠ 記事種別（D列）とバナー種別（G列）は手入力。 スクリプト更新後に消えないよう、スクリプトは空白で書き込み＆列を保持する設計にしてある。

色の意味

薄い緑 = PU＋バナー 両方設置済み
薄い青 = PUのみ設置済み
オレンジ = 未設置（標準）
赤みがかった = 未設置・最優先（あいたタブ）
グレー = CV記事（LINE対象外、こいずみタブ）

スプシ更新コマンド

dashboard/ 以下のスクリプトを個別に実行する。所要時間は各タブ約10秒。

# カレントディレクトリを移動 cd ~/OneDrive/デスクトップ/project-noa/dashboard # タブを個別更新（対象タブだけ実行してOK） python3 add_aita_sheet.py # あいた（いびき） python3 add_oda_sheet.py # 小田（脱毛＋ダイエット） python3 add_shibuya_sheet.py # 渋谷（ダイエット） python3 add_koizumi_sheet.py # こいずみ（ダイエット） python3 add_mega_sheet.py # メガ
ℹ️ スクリプトは friend_data_cache.csv を読んでPU/バナー設置状況を判定する。キャッシュデータが古い場合は更新が必要。スクリプトの場所: project-noa/dashboard/
